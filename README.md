# PARLA CHIARO — Dialect-Aware Warning System (DAWS)

[![Python 3.10+](https://img.shields.io/badge/python-3.10+-blue.svg)](https://www.python.org/)
[![Ollama](https://img.shields.io/badge/LLM-Mistral%207B%20v0.3-orange.svg)](https://ollama.com/)
[![ASR](https://img.shields.io/badge/ASR-WhisperX%20Large%20V3-red.svg)](https://github.com/m-bain/whisperX)

**PARLA CHIARO** is a local-first **Uncertainty Quantification (UQ)** framework for cascaded voice pipelines (Speech-to-Text → LLM) in the healthcare domain. The system is specifically designed to intercept and halt the propagation of diagnostic errors induced by the phonetic and semantic distortions of **Neapolitan dialect**.

The architecture implements the **1D Spectral Markov** method, capable of operating in fully online, single-pass mode at runtime (no Ground Truth required), outperforming the offline state-of-the-art *Inv-Entropy* (Song et al., NeurIPS 2025) on clinical output quality metrics.

---

## 🛠️ System Architecture

The framework is organised into four sequential macro-modules:

1. **ASR & Test-Time Data Augmentation (TTA):** WhisperX Large V3 performs the primary transcription. A Gaussian acoustic perturbation module ($\alpha=0.005$) generates two live audio variants to quantify aleatoric speaker uncertainty.
2. **1D Geometric SVM:** 768D SBERT embeddings are projected onto separate drift axes for Input (acoustic) and Output (clinical), reducing computational complexity to $O(N^2)$ online.
3. **Spectral Markov Core:** Asymmetric causal modelling via coupling of stochastic matrices generated by Laplacian kernels ($k=1$). The spectral entropy $H_{\text{spectral}}$ is extracted from the modulus of the complex eigenvalues of the joint product $P_y P_x$.
4. **Triage Dashboard:** Streamlit frontend mapping clinical risk via a multiplicative Severity Score clipped against outliers.

## 🚀 Quick Start

The project is configured to run entirely on local hardware — Apple Silicon (MPS acceleration) or NVIDIA GPU.

### 1. Prerequisites

* **Python 3.10+**
* **MongoDB** (running in background on default port `27017`)
* **Ollama** with Mistral loaded:
  ```bash
  ollama run mistral
  ```

### 2. Clone and Set Up Environment

```bash
git clone https://github.com/marcociama/parla-chiaro.git
cd parla_chiaro

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install -r requirements.txt
```

### 3. Offline Calibration (N=50)

Before running production, calibrate the triage thresholds on the 50-patient corpus:

```bash
python scripts/benchmark_final.py
```

This generates `config/geometry_calibration.json` with the empirical entropy bounds and drift axis vectors.

### 4. Launch the Live Dashboard

```bash
streamlit run daws/ui/app.py
```
